:::tip Gitee Парадигма формы - это грамматическая спецификация, которая структурно описывает определение формы. Может использоваться для настройки [функции шаблона Gitee Issue](./issue/template.mdx)。
:::

В этом документе будет описано, как определить и настроить поля формы, использующей Gitee

## Определение элемента шаблона формы

Для элементов формы вы можете задать следующие свойства:

| ключ | описание | назначение | тип | Значение по умолчанию | Допустимое значение |
|:-:|:--|:-:|---|---|---|
| `type` | Тип элемента | Обязательный | строка | - | Смотрите инструкции ниже |
| `id` | Идентификатор элемента，`type` для `markdown` За исключением тех случаев, когда <br />`id` Можно использовать только буквенно-цифровые символы、`-` и `_`。`id` Должно быть уникальным в определении формы, если `id` установлен，`id` Является стандартным идентификатором поля в предварительном заполнении параметра URL-запроса | необязательный | строка | - | - |
| `attributes` | Пары ключ-значение используются для описания и определения атрибутов конкретных элементов формы | Обязательный | Отображение полей | - | - |
| `validations` | Пары ключ-значение, используемые для установки ограничений на определенные элементы формы | необязательный | Отображение полей | - | - |

В качестве значения параметра `type`  вы можете выбрать один из следующих типов элементов формы (form items). Каждый тип имеет уникальные атрибуты и проверки.

| тип | описание |
|---|---|
| [`checkboxes`](#checkboxes) | 一Флаг группы |
| [`dropdown`](#dropdown) | Выпадающее меню |
| [`input`](#input) | Однострочное текстовое поле |
| [`markdown`](#markdown) | Markdown предоставляет пользователю дополнительные возможности **форматирования** |
| [`textarea`](#textarea) | Многострочное текстовое поле |

## `attributes` и `validations` определение {#attributes-and-validations}

对于 `attributes` и `validations`，Соответствующие определения относятся к следующему：

## `markdown` Элемент формы {#markdown}

:::info Описание элемента
Вы можете использовать `value` элемента `markdown` для отображения содержимого Markdown в форме, чтобы предоставить пользователям дополнительный контекст, но содержимое `value` по умолчанию не передается и будет отправлено только после того, как пользователь изменит его.
:::

### Определение атрибутов {#markdown-attributes}

| ключ | описание | Обязательный | тип | Значение по умолчанию | Допустимое значение  |
| --- | --- | --- | --- | --- | --- |
| `value` | Отрисованный текст. Поддерживает формат Markdown | Обязательное поле | строка | - | - |

:::info
Обработка YAML обрабатывает хэш-символы как комментарии. Чтобы вставить заголовок Markdown, заключите текст в кавычки. Для многострочного текста вы можете использовать оператор вертикальной строки (Обратитесь к синтаксису YAML)
:::

Ниже приведен конкретный пример YAML：

```yaml showLineNumbers
body:
- type: markdown
  value: "## Спасибо за Ваш вклад в наш проект!"
- type: markdown
  attributes:
   value: |
  Спасибо за то, что Вы нашли время заполнить отчет об ошибке.
```

## `textarea` Текстовое поле {#textarea}

:::info Описание элемента
Вы можете использовать элемент `textarea` для добавления многострочных текстовых полей в форму. Участники также могут прикреплять файлы в поле `textarea`
:::

### attributes определение {#textarea-attributes}

| ключ | описание | обязательный | тип | Значение по умолчанию | Допустимое значение  |
| --- | --- | --- | --- | --- | --- |
| `label` | Краткое описание ожидаемого пользовательского ввода, также отображается в виде формы | обязательный | строка | - | -
| `description` | Описание текстовой области, содержащее контекст или рекомендации, отображаемое в виде формы | Необязательное поле | строка | Пустая строка | -
| `placeholder` | Полупрозрачные заполнители, отображаемые, когда текстовая область пуста | Необязательное поле | строка | Пустая строка | -
| `value` | Текст, предварительно заполненный в текстовой области | Необязательное поле | строка | - | -
| `render` | Если указано значение, то отправленный текст будет отформатирован в виде блока кода. Если нет, то текстовая область будет расширена для вложений файлов или редактирования с помощью Markdown | Необязательное поле | строка | - | Известные типы языков.Для получения дополнительной информации, пожалуйста, обратитесь к [Объявлению определения типа языка] (https://github.com/github/linguist/blob/master/lib/linguist/languages.yml)  |

### validations определение {#textarea-validations}

| ключ | описание | обязательный | тип | Значение по умолчанию | Допустимое значение |
| - | --- | --- | --- | --- | --- |
| `required` | Предотвратите отправку формы до того, как элемент будет заполнен. Применимо только к общедоступным репозиториям | Необязательное поле | Логические значения | false | - |

Ниже приведен конкретный пример YAML：

```yaml showLineNumbers
body:
- type: textarea
  id: repro
  attributes:
   label: Этапы воспроизведения
   description: "Как вы вызвали эту ошибку? Пожалуйста, расскажите нам шаг за шагом"
   value: |
  1.
  2.
  3.
  ...
   render: bash
  validations:
   required: true
```

## `input` Поле ввода {#input}

:::info Описание элемента
Вы можете использовать `input` . Элемент добавляет однострочное текстовое поле（Input Поле ввода）к форме
:::

### attributes определение {#input-attributes}

| ключ | описание | обязательный | тип | Значение по умолчанию | Допустимое значение  |
| --- | --- | --- | --- | --- | --- |
| `label` | Краткое описание ожидаемого пользовательского ввода также отображается в виде формы | Обязательное поле | строка | - | -
| `description` | Описание поля, содержащее контекст или рекомендации, отображаемое в виде формы | Необязательное поле | строка | Пустая строка | -
| `placeholder` | Полупрозрачные заполнители, отображаемые, когда поле пусто | Необязательное поле | строка | Пустая строка | -
| `value` | Предварительно заполненный текст в поле | Необязательное поле | строка | - | -

### validations определение {#input-validations}

| ключ | описание | Обязательное поле | тип | Значение по умолчанию | Допустимое значение |
| - | --- | --- | --- | --- | --- |
| `required` | Предотвратите отправку формы до того, как элемент будет заполнен. Применимо только к общедоступным репозиториям | Необязательное поле | Логические значения | false | - |

Ниже приведен конкретный пример YAML：

```yaml showLineNumbers
body:
- type: input
  id: prevalence
  attributes:
   label: Распространённость ошибки
   description: "Насколько часто Вы или знакомые Вам люди сталкиваются с этой ошибкой?"
   placeholder: "Например: Я вижу эту ошибку каждый раз, когда захожу в профиль пользователя (пару раз в неделю)"
  validations:
   required: true
```

## `dropdown` Выпадающий список {#dropdown}

:::info Описание элемента
Вы можете использовать элемент `dropdown`, чтобы добавить выпадающее меню на форму
:::

### attributes определение {#dropdown-attributes}

| ключ | описание | Обязательное поле | тип | Значение по умолчанию | Допустимое значение  |
| --- | --- | --- | --- | --- | --- |
| `label` | Краткое описание ожидаемого ввода данных пользователем отображается в виде формы | Обязательное поле | строка | - | -
| `description` | Описание выпадающего списка, содержащее контекст или рекомендации, отображаемое в виде формы | Необязательное поле | строка | Пустая строка | -
| `multiple` | Определите, может ли пользователь выбрать несколько вариантов | Необязательное поле | Логические значения | false | -
| `options` | Множество опций, из которых пользователь может выбирать. Не может быть пустым, все варианты должны быть разными | Обязательное поле | Массив строк | - | -

### validations определение {#dropdown-validations}

| ключ | описание | Обязательное поле | тип | Значение по умолчанию | Допустимое значение |
| - | --- | --- | --- | --- | --- |
| `required` | Предотвратите отправку формы до того, как элемент будет заполнен. Применимо только к общедоступным репозиториям | Необязательное поле | Логические значения | false | - |

Ниже приведен конкретный пример YAML：

```yaml showLineNumbers
body:
- type: dropdown
  id: download
  attributes:
   label: Как Вы загрузили программу?
   options:
  - Homebrew
  - MacPorts
  - apt-get
  - Собрал из исходников
  validations:
   required: true
```

## `checkboxes` 复选框 {#checkboxes}

:::info Описание элемента
Вы можете использовать элемент `checkboxes`, чтобы добавить набор флажков на форму
:::

### attributes определение {#checkboxes-attributes}

| ключ | описание | Обязательное поле | тип | Значение по умолчанию | Допустимое значение  |
| --- | --- | --- | --- | --- | --- |
| `label` | Краткое описание ожидаемого ввода данных пользователем отображается в виде формы | Обязательное поле | строка | - | -
| `description` | Описание установленного флажка отображается в виде формы. Поддерживает формат Markdown | Необязательное поле | строка | Пустая строка | -
| `options` | Набор флажков, которые пользователь может установить. Что касается грамматики, смотрите ниже | Обязательное поле | массив | - | -

Для каждого значения в массиве `options` могут быть установлены следующие ключи

| ключ | описание | Обязательное поле | тип | Значение по умолчанию | Допустимое значение  |
| --- | --- | --- | --- | --- | --- |
| `label` | Краткое описание ожидаемого ввода данных пользователем отображается в виде формы | Обязательное поле | строка | - | -
| `description` | Описание установленного флажка отображается в виде формы. Поддерживает формат Markdown | Необязательное поле | строка | Пустая строка | -
| `options` | Набор флажков, которые пользователь может установить. Что касается грамматики, смотрите ниже | Обязательное поле | массив | - | -

Для каждого значения в массиве `options` могут быть установлены следующие ключи

| ключ | Значение | Обязательное поле | тип | Значение по умолчанию | Вариант |
|:-:|:--|:-:|:-:|:-:|:-:|
| label | Идентификатор опции отображается в форме. Поддерживает Markdown для форматирования текста жирным шрифтом или курсивом и гиперссылок на текст | Обязательное поле | Строка | - | - |

### validations определение {#checkboxes-validations}

| ключ | описание | Обязательное поле | тип | Значение по умолчанию | Допустимое значение |
| - | --- | --- | --- | --- | --- |
| `required` | Предотвратите отправку формы до того, как элемент будет заполнен. Применимо только к общедоступным репозиториям | Необязательное поле | Логические значения | false | - |

Ниже приведен конкретный пример YAML：

```yaml showLineNumbers
body:
- type: checkboxes
  id: operating-systems
  attributes:
   label: Какие операционные системы Вы использовали?
   description: Вы можете выбрать несколько
   options:
  - label: macOS
  - label: Windows
  - label: Linux
```
